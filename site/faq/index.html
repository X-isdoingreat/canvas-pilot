Canvas Pilot FAQ

What is Canvas Pilot?

Canvas Pilot is a local-first Canvas LMS AI agent that learns recurring course assignment patterns and reuses them through scan, approval, and execute workflows.

How is it different from Canvas MCP?

Canvas MCP servers expose Canvas tools to an agent. Canvas Pilot adds recurring course workflow memory, approval boundaries, verification, and run reports above Canvas access.

Can it write?

The important distinction is not whether an agent can write. The product strength is that repeated course patterns become reusable workflows with approval and review boundaries.

Who should use it?

It is best for AI power users who already use local agent tools such as Codex or Claude Code and whose Canvas workload repeats weekly.

Does it auto-submit?

The default behavior is draft production plus student review. Any submit behavior must be explicitly authorized for the exact workflow and guarded by verification.

Where does private course data go?

Credentials, cookies, school-specific course identifiers, assignment sources, drafts, and run outputs stay on the user's machine.
